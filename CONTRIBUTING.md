# Contributing to VC 2.0 Essentials

Thank you for your interest in contributing.

## How to Contribute

### Bug Reports

If you find a bug in the financial engines or methodology documentation, please open an issue with:

- A clear description of the problem
- Steps to reproduce (for engine bugs)
- Expected vs. actual behavior
- Your Python version and OS

### Engine Improvements

The Safer Scenario Simulator and Monte Carlo engine are the core of this project. Contributions that improve accuracy, performance, or output quality are welcome. Please:

1. Fork the repository
2. Create a feature branch
3. Add tests if applicable
4. Submit a pull request with a clear description

### Methodology Feedback

The scorecard and Safer methodology are based on *Venture Capital 2.0* by John Cowan. If you have substantive feedback on the frameworks, please open a discussion rather than a PR — methodology changes need to align with the book.

### Prompt Improvements

Better prompts that work across more AI platforms are always welcome. If you've tested a prompt variation that produces better results on a specific platform, please share it.

## What We Don't Accept

- Changes to the Premium upsell or licensing terms
- Additions that duplicate Premium-only features (thesis development, capital structuring, VREC governance, etc.)
- Telemetry or tracking code

## Code of Conduct

Be respectful, constructive, and focused on improving the tools for the venture building community.
